Split by purpose — Dashboard + Profile
Two portal pages with clearly different jobs. Dashboard is your private "how am I doing" page (stats, gamification, membership). Profile is the "what others see + controls" page. Public page becomes one unified `/profile/[slug]` that honors visibility toggles.
Pros · clearest mental model
· taste data lives in the editor
· public profile is finally complete
· kills `/builders/[slug]` duplication
Cons · two portal pages — avatar click picks one
· biggest user-facing change
· more routing

C
Mirror mode — private IS public + controls
The private profile page renders exactly what the public page renders. Each section has an inline edit button and an eye toggle for `show_*`. WYSIWYG — zero gap between "what I see" and "what they see". Stats/gamification move to a separate `/portal/dashboard`.
Pros · no mental translation — spatial controls
· visibility becomes obvious, not hidden in a panel
· private/public stay in sync by design
Cons · biggest redesign lift
· still needs `/portal/dashboard` split
· some sections don't fit the pattern cleanly
